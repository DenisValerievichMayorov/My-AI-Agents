 Memory for LLMs is still a new category, and unlike traditional databases, there's no one-size-fits-all approach. 
 When you're building a memory layer, you'll need to make a few decisions: 
 - Should you use User memory or Agent memory? 
- Vector search or Graph memory for relationships? 
- How long should memories persist? 
- What metadata structure makes sense for your use case? 
 We've built the primitives so you can design the right solution for your needs, but that also means there are real choices to be made. 
 That's where our Cookbooks come in. Think of them as guiding principles while you navigate these decisions. Each one walks through the trade-offs for different use cases with real implementation examples. 
 Browse the cookbooks: https://docs.mem0.ai/cookbooks/overview 
 Building memory is as much about design as it is about code. These guides help you think through both.
