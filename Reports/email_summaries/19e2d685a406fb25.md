Hi there 
 Four weeks ago we shipped the token-efficient memory algorithm and called out where it fell short: temporal reasoning, multi-session reasoning, and event ordering. 
 This week, we released two new capabilities to close that gap, on the same &lt;7,000 token retrieval budget. 
 - Temporal Reasoning Every memory now carries a time signature. Search understands what's current, what's historical, and what's upcoming. State keys link evolving facts (jobs, preferences, relationships) so the timeline stays clean. 
- Memory Decay Recently used memories get a soft boost, up to 1.5x. Idle ones get gently dampened, down to 0.3x. Nothing is deleted or hidden. A stale-but-relevant memory still surfaces when it's the best match. 
 What the numbers look like: 
 - LongMemEval: 93.4% → 94.4% 
- LoCoMo: 91.6% → 92.5% 
- LongMemEval temporal reasoning: +3.8 points 
- LoCoMo open-domain: +6.3 points 
 A few examples of what this would fix: a coding agent resurfacing an old side project when asked about "this week's work". A personal assistant citing a dietary preference the user changed a month ago. A CRM agent quoting terms that were upgraded two months ago. The memories were already being stored correctly, but the retrieval now has a sense of time. 
 The two new features are platform-only for now. Temporal Reasoning is on by default for new projects. Memory Decay can be toggled from the dashboard or SDK.
