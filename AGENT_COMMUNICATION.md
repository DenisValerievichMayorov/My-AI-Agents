## Правила коммита и публикации на GitHub
Агенты должны самостоятельно фиксировать и публиковать завершенные изменения, если задача явно выполнена и рабочее дерево находится в репозитории `DenisValerievichMayorov/My-AI-Agents`.

### Проверка перед коммитом
1. Убедиться, что текущий каталог репозитория правильный:
   ```bash
   git remote -v
   git status --short --branch
   ```
2. Работать с репозиторием `~/Sync` / `C:\Users\anton\Sync`, а не с домашним каталогом пользователя.
3. Не добавлять временные файлы Syncthing и системные файлы:
   - `~syncthing~*.tmp`
   - `*.sync-conflict-*`
   - файлы профиля Windows/Linux, если они случайно попали в статус
4. Перед коммитом просмотреть изменения:
   ```bash
   git diff --stat
   git diff
   ```

### Коммит
1. Добавлять только релевантные файлы задачи:
   ```bash
   git add <files>
   ```
   Если задача охватывает весь текущий набор изменений и нет мусорных файлов, допустимо:
   ```bash
   git add -A
   git reset -- '~syncthing~*.tmp'
   ```
2. Сообщение коммита писать коротко, в стиле Conventional Commits:
   - `feat: ...`
   - `fix: ...`
   - `docs: ...`
   - `chore: ...`
3. Создать коммит:
   ```bash
   git commit -m "feat: краткое описание изменения"
   ```

### Публикация
1. После успешного коммита проверить, что ветка впереди `origin/main`:
   ```bash
   git status --short --branch
   ```
2. Опубликовать:
   ```bash
   git push origin main
   ```
3. После push проверить чистое состояние:
   ```bash
   git status --short --branch
   ```
   Ожидаемый результат: `main...origin/main` без `ahead` и без незакоммиченных изменений.

### Если push не проходит
- Если требуется авторизация GitHub, использовать системный Git Credential Manager или уже настроенный токен.
- Не менять remote URL без явной причины.
- Не выполнять `git reset --hard`, `git checkout -- <file>` или удаление файлов для "очистки" без явного разрешения Дениса.
- Если есть конфликт с удаленной веткой, сначала выполнить:
  ```bash
  git pull --rebase origin main
  ```
  затем повторить `git push origin main`.
