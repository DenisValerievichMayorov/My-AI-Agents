## 1. Архитектура Нашего Взаимодействия (Communication Protocol)

Мы общаемся и координируем задачи асинхронно через:
1. **`GEMINI.md`** (в корне вашего профиля): Наша общая долговременная память, история изменений и лог активностей. Каждый раз, начиная сессию, читай её, чтобы узнать текущие задачи, и обязательно дописывай свои успехи в "Историю изменений".
2. **Папка `~/Sync`**: Синхронизируемое через Syncthing хранилище, где лежат общие скрипты автоматизации (`~/Sync/Scripts`), база знаний (`~/Sync/knowledge_base`), и отныне — конфигурации самого Pi!

## 2. Мгновенная Автоматическая Настройка (Autopilot Setup)

Чтобы твое окружение стало абсолютно синхронным с Windows PC, я написал автоматический bash-скрипт. Тебе достаточно запустить в терминале всего одну команду:

```bash
sh ~/Sync/Scripts/setup_pi_termux_chromebook.sh
```

### Что сделает этот скрипт:
1. **Свяжет директорию настроек Pi:** Перенаправит локальную папку `~/.pi` на синхронизируемую папку `~/Sync/.pi` через символическую ссылку. Теперь все установленные модели, сессии и кастомные расширения будут общими!
2. **Импортирует правила поведения (.clinerules):** Твои локальные `.clinerules` и `.cursorrules` будут ссылаться на правила из `Sync`. Ты сразу начнешь использовать **Architect Mode Protocol** (планирование, самовосстановление, делегирование).
3. **Настроит переменные среды:** Пропишет петлевой локальный адрес `export OLLAMA_HOST=127.0.0.1:11434` в `.bashrc`/`.zshrc` (на Android соединение с `0.0.0.0` запрещено системой).
4. **Создаст быстрый алиас:** Добавит команду `pi-sync` для быстрого обновления настроек.

## 3. Твои новые суперспособности (Custom Extensions & Tools)

После запуска скрипта в твоем арсенале появятся новые мощные инструменты, которые я разработал:

1. **Мультиагентный опрос (`parallel-agents.ts`):** 
   * Инструмент: `parallel_agent_query`
   * Позволяет тебе опрашивать параллельно или по цепочке fallback локальную Ollama, облачный OpenRouter (с бесплатной Qwen3 Coder) и других CLI-агентов, возвращая единый структурированный отчет.
2. **Интеграция Google Workspace (`google-tools.ts`):** 
   * Инструменты: `google_gmail_search`, `google_calendar_list`, `google_drive_list`
   * Позволяет тебе напрямую искать письма в Gmail, просматривать события календаря и получать списки файлов на Google Диске через OAuth-токен в `google_token.json` (работает полностью автономно и локально через скрипт `google_tool.py`).

## 4. Чек-лист для запуска Ollama на телефоне (Termux)
Если ты планируешь запускать локальные LLM-модели на процессоре телефона:
1. Установи Ollama в Termux: `pkg install ollama`
2. Запусти сервер в фоновом режиме: `ollama serve &`
3. Скачай легковесную модель для работы (например, `gemma2:2b` или `llama3.2`): `ollama run gemma2:2b`

*Если локальная Ollama не отвечает или модель не скачана, твое кастомное расширение автоматически переключится на **Qwen3 Coder от OpenRouter**!*
