# Objective
Организовать синхронизацию папки Sync (которая уже синхронизируется через Syncthing на 3 устройствах) с Google Drive.

# Context, Memory & Existing Tools
- Имеется установленный клиент Google Drive for Desktop (в `C:\Program Files\Google\Drive File Stream`).
- Имеется папка `C:\Users\anton\Sync`.
- Syncthing отлично работает для P2P-синхронизации, но Google Drive требует либо официального клиента, либо утилиты `rclone` для надежной передачи файлов.
- Создание велосипеда на Node.js чревато конфликтами версий и затиранием файлов при двусторонней синхронизации.

# Step-by-Step Strategy
1. **Google Drive Desktop (Основной способ - Real-time):** Запустить официальный клиент (сделано). Пользователю нужно будет авторизоваться и добавить папку через интерфейс. Это обеспечит двустороннюю, надежную синхронизацию в реальном времени.
2. **Rclone (Альтернатива - по расписанию):** Скачать `rclone.exe` в папку `agent_tools`. Создать скрипт для авторизации и настройки резервного копирования в Google Drive (однонаправленная или двунаправленная синхронизация раз в N часов).

# Tool Creation
Скачан `rclone.exe` в `C:\Users\anton\agent_tools\`. Создан вспомогательный скрипт `C:\Users\anton\agent_tools\rclone_sync_setup.bat`.
